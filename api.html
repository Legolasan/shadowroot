Linked In API Reference

Ad Analytics API's

Retention

The retention period for data depends on the timeGranularity of your request and type of request (demographics vs. non-demographic analytics):

Freequency Demographic Non-Demographic
DAILY Last 6 months Last 10 years
MONTHLY Last 2 years Last 10 years
YEARLY Last 2 years Last 10 years
ALL Last 2 years Last 10 years

Restrictions

Other restrictions apply based on your query parameters:

  • When timeGranularity=ALL, if either the start or end date of a request range is outside of the 6 month daily retention period, the dates will be automatically rounded to month boundaries, and the entire month's data is included in the response. For example, if a request is made on 8/1/2017 with dateRange.start = 1/15/2017 and dateRange.end = 7/15/2017, then dateRange.start will be automatically adjusted to 1/1/2017. This is because 1/15/2017 is outside of a 6 month retention period for daily data, whereas monthly data for January is still within the 2 year retention period.
  • Demographic pivots does not support carousel and video ads metrics.

Delays

Performance metrics (e.g. metrics about a creative, campaign, or account) are near realtime. Metrics based on demographics (Company Size, Job Function, etc) may be delayed by a couple hours.

Accuracy

To protect LinkedIn member privacy, demographic metrics are approximate. The demographic metrics will be within 3 units (impressions, clicks, or conversions) of actual results in almost all cases.

Because metrics are approximate for privacy protection, you may observe minor inconsistencies when:

  1. comparing similar time ranges, for example, reporting on results over 7 days vs. 8 days
  2. comparing the sum of daily metrics to the full corresponding time period
  3. comparing reporting levels, for example, adding campaign level data and comparing it to account level data

To minimize the effect of privacy approximation, select:

  1. a full time period instead of adding up daily metrics for that time period
  2. the highest reporting level you want to view, for example, select account level data instead of adding up campaign level data

Usage

To use the Ad Analytics API, we will need to provide a few query parameters in the analytics Finder. The parameter q=analytics or q=statistics is always required.

GET https://api.linkedin.com/v2/adAnalyticsV2?q=analytics&pivot={pivot}&timeGranularity={time} Copy

Query Parameters

Query Parameters Description Format Required
q Designates the query finder. The only allowed values are "analytics" and "statistics". Use "analytics" to specify a single pivot. Use "statistics" to specify up to two pivots. STRING Yes
pivot, pivots

Pivot of results, which each report data point is grouped by. Use "pivot" for query finder type "analytics". Use "pivots" for query finder type "statistics".

The below are valid enum values supported by both the "analytics" and "statistics" query finder:

  • COMPANY - Group results by advertiser's company.
  • ACCOUNT - Group results by account.
  • SHARE - Group results by sponsored share.
  • CAMPAIGN - Group results by campaign.
  • CREATIVE - Group results by creative.
  • CAMPAIGN_GROUP - Group results by campaign group.
  • CONVERSION - Group results by conversion.
  • SERVING_LOCATION - Group results by serving location, onsite or offsite.
  • CARD_INDEX - Group results by the index of where a card appears in a carousel ad creative. Metrics are based on the index of the card at the time when the user's action (impression, click, etc.) happened on the creative (Carousel creatives only).

The below demographics enum values are supported only by the "analytics" finder with a single pivot:

  • MEMBER_COMPANY_SIZE - Group results by member company size.
  • MEMBER_INDUSTRY - Group results by member industry.
  • MEMBER_SENIORITY - Group results by member seniority.
  • MEMBER_JOB_TITLE - Group results by member job title.
  • MEMBER_JOB_FUNCTION - Group results by member job function.
  • MEMBER_COUNTRY - Group results by member country.
  • MEMBER_REGION - Group results by member region.
  • MEMBER_COMPANY - Group results by member company.
 String Yes
dateRange.start Represents the inclusive start time range of the analytics. If unset, it will indicate an open range up to the end time. Date Object Yes
dateRange.end Represents the inclusive end time range of the analytics. Must be after start time if it's present. If unset, it will indicate an open range from start time to everything after. Date Object No
timeGranularity

Time granularity of results. Valid enum values:

  1. ALL - Results grouped into a single result across the entire time range of the report.
  2. DAILY - Results grouped by day.
  3. MONTHLY - Results grouped by month.
  4. YEARLY - Results grouped by year.
 String Yes
campaignType Match result by a campaign type. Supported types are [TEXT_AD, SPONSORED_UPDATES, SPONSORED_INMAILS, DYNAMIC]. Requires at least one other facet. Defaults to empty. Array of Campaign Type Values No
shares Match result by share facets. Defaults to empty. Array ofShare URN No
campaigns Match result by campaign facets. Defaults to empty. Array ofSponsored Campaign URN No
creatives Match result by creative facets. Defaults to empty. Array ofSponsored Creative URN No
campaignGroups Match result by campaign group facets. Defaults to empty. Array of Campaign Group URN No
accounts Match result by sponsored ad account facets. Defaults to empty. Array of Account URN No
companies Match result by company facets. Defaults to empty. Array of Company URN No

Sample request to query for analytics from January 1, 2016 in campaign id 119972645 with a monthly time granularity and pivoting based on creatives:

GET https://api.linkedin.com/v2/adAnalyticsV2?q=analytics&dateRange.start.month=1&dateRange.start.day=1&dateRange.start.year=2016&timeGranularity=MONTHLY&pivot=CREATIVE&campaigns[0]=urn:li:sponsoredCampaign:119972645 Copy

AdAnalytics Schema

Field Name

Type

Description

actionClicks

optional long

  • The count of clicks on the action button of the Sponsored InMail. Sponsored InMail only.

adUnitClicks

optional long

  • The count of clicks on the ad unit displayed alongside the Sponsored InMail. Sponsored InMail only.

cardClicks

optional long

  • The number of clicks for each card of a carousel ad. The first card click of the carousel ad results in an immediate cardClick and click, whereas scrolling to other cards and clicking will count as additional cardClick.

cardImpressions

optional long

  • The number of impressions shown for each card of a carousel ad. The first card of the carousel ad results in an immediate cardImpression and impression, whereas scrolling to other cards will count as additional cardImpressions.

clicks

long

  • The count of chargeable clicks. Despite not charging for clicks for CPM campaigns, this field still represents those clicks for which we would otherwise charge advertisers (i.e. clicks to view the landing page or company page).

commentLikes

optional long

  • The count of likes of a comment. Sponsored Updates only.

comments

optional long

  • The count of comments. Sponsored Updates only.

companyPageClicks

optional long

  • The count of clicks to view the company page.

conversionValueInLocalCurrency

optional BigDecimal

  • Value of the conversions in the account's local currency based on rules defined by the advertiser. Conversion value is set by the advertiser at a per conversion level, and aggregated across the query time range.

costInLocalCurrency

optional BigDecimal

  • Cost in the account's local currency based on the pivot and timeGranularity. For example, this would be spend by member company size per month if the pivot is MEMBER_COMPANY_SIZE and timeGranularity is MONTHLY.

costInUsd

optional BigDecimal

  • Cost in USD based on the pivot and timeGranularity. For example, this would be spend by campaign on a given day if the pivot is CAMPAIGN and timeGranularity is DAILY.

dateRange

DateRange

  • Date range covered by the report data point. Date is specified in UTC. Start and end date are inclusive. Start date is required. End date is optional and defaults to today

externalWebsiteConversions

long, default="0"

  • The count of conversions indicated by pixel loads on an external advertiser website.

externalWebsitePostClickConversions

long, default="0"

  • The count of post-click conversions indicated by pixel loads on an external advertiser website.

externalWebsitePostViewConversions

long, default="0"

  • The count of post-view conversions indicated by pixel loads on an external advertiser website.

follows

optional long

  • The count of follows. Sponsored Updates only.

fullScreenPlays

optional long

  • A user taps on the video, going into video view mode.

impressions

long

  • This is the count of 'impressions' for Direct Ads and Sponsored Updates and 'sends' for InMails.

landingPageClicks

long

  • The count of clicks which take the user to the creative landing page.

leadGenerationMailContactInfoShares

optional long

leadGenerationMailInterestedClicks

optional long

likes

optional long

  • The count of likes. Sponsored Updates only.

oneClickLeadFormOpens

optional long

  • The count of times a user opened the lead form for a One Click Lead Gen campaign.

oneClickLeads

optional long

  • The count of leads generated through One Click Lead Gen.

opens

optional long

  • The count of opens of Sponsored InMail. Sponsored InMail only.

otherEngagements

long

  • The count of user interactions with the ad unit that do not fit into any other more specific category.

pivotValues

string[], default="[]"

  • The value of the pivots for a specific record returned. For example, supplying pivots of CREATIVE and CONVERSION will result in a list of records, one for each creative/conversion combination. The pivotValues would contain serialized urns for the specific creative and conversion for a record.

sends

optional long

  • The count of sends of Sponsored InMail. Sponsored InMail only.

shares

optional long

  • The count of shares. Sponsored Updates only.

textUrlClicks

optional long

  • The count of clicks on any links (anchor tags) that were included in the body of the Sponsored InMail. Sponsored InMail only.

totalEngagements

long

  • The count of all user interactions with the ad unit.

videoCompletions

optional long

  • The count of video ads that played 97-100% of the video. This includes watches that skipped to this point if the serving location is ON_SITE.

videoFirstQuartileCompletions

optional long

  • The count of video ads that played through the first quartile of the video. This includes watches that skipped to this point if the serving location is ON_SITE.

videoMidpointCompletions

optional long

  • The count of video ads that played through the midpoint of the video. This includes watches that skippped to this point if the serving location is ON_SITE.

videoStarts

optional long

  • The count of video ads that were started by a user.

videoThirdQuartileCompletions

optional long

  • The count of video ads that played through the third quartile of the video. This includes watches that skipped to this point if the serving location is ON_SITE.

videoViews

optional long

  • A video ad playing for at least 2 continuous seconds 50% in-view, or a click on the CTA, whichever comes first. An interaction with the video (like going to fullscreen mode) does not count as a view.

viralCardClicks

optional long

  • The number of viralClicks for each card of a carousel ad. The first viralCardClick of the carousel ad results in an immediate viralCardClick and viralClick, whereas scrolling to other cards and clicking will count as additional viralCardClick.

viralCardImpressions

optional long

  • The number of viralImpressions shown for each card of a carousel ad. The first card of the carousel ad results in an immediate viralCardImpression and viralImpression, whereas scrolling to other cards will count as additional viralCardImpressions.

viralClicks

optional long

  • The count of clicks on viral impressions. See viral impressions definition. Sponsored Updates only.

viralCommentLikes

optional long

  • The count of likes on comments from viral impressions for this activity. See viral impressions definition. Sponsored Updates only.

viralComments

optional long

  • The count of comments from viral impressions for this activity. See viral impressions definition. Sponsored Updates only.

viralCompanyPageClicks

optional long

  • The count of clicks to view the company page from viral impressions for this activity. See viral impressions definition. Sponsored Updates only.

viralExternalWebsiteConversions

optional long

  • The count of conversions indicated by pixel loads on an external advertiser website driven by a viral event. See viral impressions definition.

viralExternalWebsitePostClickConversions

optional long

  • The count of post-click conversions indicated by pixel loads on an external advertiser website driven by a viral click. See viral impressions definition

viralExternalWebsitePostViewConversions

optional long

  • The count of post-view conversions indicated by pixel loads on an external advertiser website driven by a viral impression. See viral impressions definition.

viralFollows

optional long

  • The count of follows from viral impressions for this activity. See viral impressions definition. Sponsored Updates only.

viralFullScreenPlays

optional long

  • A user taps on the video, going into video view mode. See viralImpressions definition.

viralImpressions

optional long

  • The count of viral impressions for this activity. Viral impressions are those resulting from a user sharing a sponsored update to their own network of connections. Viral impressions are not counted as regular impressions. Sponsored Updates only.

viralLandingPageClicks

optional long

  • The count of clicks on viral impressions to take the user to the creative landing page. See viral impressions definition. Sponsored Updates only.

viralLikes

optional long

  • The count of likes from viral impressions for this activity. See viral impressions definition. Sponsored Updates only.

viralOneClickLeadFormOpens

optional long

  • The count of times a user opened the lead form for viral impressions from a Lead Gen campaign. See viral impressions definition.

viralOneClickLeads

optional long

  • The count of leads generated through One Click Lead Gen from viral impressions for this activity. See viral impressions definition.

viralOtherEngagements

optional long

  • The count of user interactions with viral impressions that do not fit into any other more specific category. See viral impressions definition. Sponsored Updates only.

viralShares

optional long

  • The count of shares from viral impressions for this activity. See viral impressions definition. Sponsored Updates only.

viralTotalEngagements

optional long

  • The count of all user interactions with a viral ad unit. See viral impressions definition. Sponsored Updates only.

viralVideoCompletions

optional long

  • The count of viral video ads that played 97-100% of the video. This includes watches that skipped to this point. See viralImpressions definition.

viralVideoFirstQuartileCompletions

optional long

  • The count of viral video ads that played through the first quartile of the video. This includes watches that skipped to this point. See viralImpressions definition.

viralVideoMidpointCompletions

optional long

  • The count of viral video ads that played through the midpoint of the video. This includes watches that skippped to this point. See viralImpressions definition.

viralVideoStarts

optional long

  • The count of viral video ads that were started by a user. See viralImpressions definition. Since viral videos are automatically played for ON_SITE, this will be the same as viralImpressions if the servingLocation is ON_SITE.

viralVideoThirdQuartileCompletions

optional long

  • The count of viral video ads that played through the third quartile of the video. This includes watches that skipped to this point. See viralImpressions definition

viralVideoViews

optional long

  • A viral video ad playing for at least 2 continuous seconds 50% in-view, or a click on the CTA, whichever comes first. An interaction with the video (like going to fullscreen mode) does not count as a view. See viralImpressions definition.

Please note: Querying analytics for deleted shares that were promoted as sponsored content is not currently supported. Since creatives for sponsored content reference a particular share, it is possible to retrieve analytics for the deleted share by requesting analytics for the creatives that sponsored the deleted share.
